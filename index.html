Dawnstar Trading — The Complete User Guide

Dawnstar is an automated, validation-first systematic trading app for stocks, forex, and crypto that one person runs on their own PC with their own broker keys — its real product is a rigor engine built to stop you fooling yourself, not a promise of profit.

Dawnstar Trading is a self-contained native Windows desktop app (current version 2.1.0) that runs an automated, multi-asset systematic trading pipeline over brokers you connect yourself. There is no cloud account, no web login, and no subscription. At its heart is a single C++ program (the "engine") that decides what to trade, wrapped in a "flight-deck for money" desktop UI you launch by double-clicking Dawnstar.exe. It trades three asset classes through the brokers' official APIs using your own keys: US stocks and ETFs via Alpaca, foreign exchange via OANDA, and crypto via Coinbase Advanced Trade. Your money and your account stay entirely at those regulated brokers — Dawnstar is "bring your own broker" software that never touches custody.

What makes Dawnstar different is that its philosophy is inverted from the rest of the industry. Most trading software sells a beautiful backtest curve going up-and-to-the-right; the dirty secret is that those curves are mostly overfit — someone tried hundreds of parameter combinations and kept the prettiest one, and it collapses when real money touches it. Dawnstar takes the opposite stance: it bakes professional-grade anti-overfitting statistics (Deflated Sharpe Ratio, Probability of Backtest Overfitting, walk-forward, Monte Carlo, combinatorial cross-validation) into the core workflow and shows them to you plainly — even when the honest answer is "this isn't real." Every backtest and every scored strategy carries a plain-English significance verdict. The honesty is deliberately the product, not the marketing.

Who it is for: a single technically-comfortable person who wants to run a disciplined, diversified, systematic book with their own capital and their own broker accounts, and who values knowing the truth about a strategy over being sold a dream. It is not for people who want a hands-off money machine, day-trading signals, or someone to manage their money — none of which Dawnstar is or pretends to be.

The validation-first philosophy shows up everywhere. The app defaults to paper/practice trading, all account data is read-only, and moving real money requires deliberately clearing two independent gates on top of supplying live keys. The strategy thesis is that edge comes from breadth — combining many genuinely uncorrelated, individually-modest bets — not from a clever signal. And the app's bedrock, repeated everywhere, is that it has "no demonstrated edge": even its statistically strongest backtest is still just a backtest with no live track record, so the correct way to use it is to deploy small, paper-track for months, and scale only if forward results confirm.

The core philosophy: built to stop you fooling yourself

This is the whole point of Dawnstar. The trading-software industry sells the dream of a backtest equity curve that only goes up. Most of those curves are overfit — the result of trying hundreds of combinations and keeping the prettiest — and they fall apart on real money. Dawnstar refuses to play that game. The tools that do this are collectively the 'Rigor Engine,' and the same C++ engine that computes them is the one that trades live, on a deterministic seed, so the scorecard matches what actually runs. As the docs put it: none of this makes a losing strategy win — that is the point. The honest verdict is the product.

The strategy thesis: breadth beats a clever signal

Dawnstar's research conclusion is that there is no magic indicator. Every real trading signal is weak — a single-market trend strategy is roughly 0.4 Sharpe, and even Warren Buffett's realized Sharpe over 40 years was about 0.79. The edge comes from BREADTH: combining many genuinely uncorrelated positive-edge bets, because portfolio Sharpe grows as (per-bet Sharpe) times the square root of the number of independent bets. So Dawnstar builds a diversified basket of literature-backed 'sleeves' rather than one strategy, and it matches the mechanism to the market. Trend-following (time-series momentum) is the unifying core because it has multi-decade, cross-asset, out-of-sample evidence across equities, FX, and crypto. Mean-reversion is kept only as an equities sleeve, gated by trend strength, and is never run on crypto or FX. Crypto trends but is punished by fees, so it gets trend/low-vol only. FX gets trend plus a small, capped, tail-aware carry sleeve. Two further principles: never chase intraday/day-trading alpha (studies find roughly 97% of persistent day traders lose), and use Kelly sizing only as a CAP (at most quarter-Kelly), never a target.

Installing and launching the desktop app

For most users the install is: get the app onto your machine and run the exe. The public website download is packaged as a single Dawnstar-Setup.exe (with a companion .sha256 checksum). The app itself runs as a portable Dawnstar.exe that must sit inside the install tree (next to the engine\ folder) so it can find the engine, your state\ folder, and logs\; to run it elsewhere, set a DAWNSTAR_ROOT environment variable pointing at the install root. On first run Windows SmartScreen will flag the exe as an unknown publisher because the public build is currently unsigned — this is expected, not a warning about Dawnstar specifically. The app opens one native 1360x900 window (minimum 1040x680) after starting a private local server on a random 127.0.0.1 port. It enforces a single instance: launching again while it is running just shows a 'Dawnstar is already running.' message.

  1. Place the Dawnstar folder wherever you like (the conventional fallback location is C:\DawnstarTrading).
  2. Run Dawnstar.exe (or the Desktop shortcut).
  3. If Windows shows 'unknown publisher,' clear it once: right-click Dawnstar.exe → Properties → tick 'Unblock' → OK, then relaunch.
  4. Optionally verify integrity: run (Get-FileHash .\Dawnstar.exe -Algorithm SHA256).Hash and compare it to the value in Dawnstar.exe.sha256.
  5. Wait for the native window to paint, then proceed to the first-run vault screen.
The public Dawnstar.exe is not yet Authenticode-signed, so SmartScreen will warn until it is signed; docs\SIGNING.md covers signing options.
On some GPU/driver combinations the app defaults to software rendering (--disable-gpu) to avoid a blank window; override via WEBVIEW2_ADDITIONAL_BROWSER_ARGUMENTS if needed.

First launch: creating your encrypted vault

The very first time you open Dawnstar it shows a 'Secure your vault' screen before you can reach the app. Here you set a master password that encrypts all your broker API keys locally. The password is stretched with Argon2id (a memory-hard key-derivation function: t=3, 128 MiB, p=1 — the same class of design password managers like Bitwarden use for a local vault), and the resulting key seals a single file, state\vault.dat, with AES-256-GCM authenticated encryption; on Windows the sealed file is additionally wrapped with DPAPI so a copy is useless on another machine or account. The master password is never stored and there is no recovery — forget it and the stored keys are gone. You must also tick a checkbox accepting the Risk Disclaimer & Terms, which explicitly acknowledges that trading risks real money, that no edge is demonstrated, and that past or backtested results never guarantee future results; the 'Create Vault & Enter' button stays disabled until you tick it, and your acceptance is timestamped to state\consent.json. Creating the vault also migrates any broker keys you had stored as Windows environment variables and deletes the plaintext originals from the registry.

  1. Type a master password (minimum 8 characters; an on-screen strength meter reads too short / weak / fair / strong / excellent).
  2. Re-type it to confirm (the two must match).
  3. Read the 'There is no recovery' warning — record this password somewhere safe offline.
  4. Tick the consent checkbox accepting the Risk Disclaimer & Terms.
  5. Click 'Create Vault & Enter →' — the app derives the key (~0.5s), seals the vault, records consent, and drops you into the dashboard.
There is genuinely no password reset or recovery — a forgotten master password means the stored keys are unrecoverable by design. Back up state\vault.dat and record the password offline before you ever store live keys.
The vault protects against a stolen laptop or a copied file, but nothing running locally can fully defend against malware already running as your Windows user — true of any desktop password manager.

Every launch after: unlocking, auto-lock, and changing the password

Once the vault exists, every launch shows a 'Welcome back' unlock screen. You type your master password and click 'Unlock' (Enter works too); a wrong password is simply rejected and the app stays sealed. While unlocked, secrets live in memory only, and the vault auto-locks after 15 minutes of inactivity, wiping them; you can also lock instantly with the sidebar '🔒 Lock Vault' button. You can change the master password any time in Settings → Security by entering the current and a new password — this re-derives the key and re-seals the vault without re-entering any broker keys. Because vault.dat is a single point of failure, Settings → Security also has a 'Back up vault' button that writes a timestamped, still-encrypted copy to state\backups.

Connect Coinbase (crypto) — the live-only exchange

Coinbase demands extra care: its Advanced Trade API has NO sandbox — there is only live api.coinbase.com. Dawnstar handles this by paper-SIMULATING crypto (logging the plan and submitting nothing) whenever the dual live-gate is not armed, so you can test safely, but there is no 'paper Coinbase' account behind it. Authentication is a modern CDP API key: a key name of the form organizations/{org}/apiKeys/{key} plus an ECDSA (EC) private key in PEM form. Dawnstar signs every request with a fresh, short-lived (2-minute) ES256 JWT — there are no HMAC secrets. Create the key at cdp.coinbase.com with least privilege (view + trade, NOT transfer/withdraw). The Portfolio Options: Sleeves, Tiers & the Honest Numbers

Dawnstar's core insight is that no single signal is strong enough to trust — the honest edge comes from combining several genuinely uncorrelated 'sleeves,' sized by risk-parity, and judging the combined stream through one strict statistical gate. The Portfolio view offers three prebuilt books.

The big idea: sleeves and breadth

A 'sleeve' is one strategy run on one asset class — for example 'equity trend on 9 sector ETFs' or 'FX carry on 10 currency pairs.' On its own, every sleeve is weak (standalone sleeve Sharpes run roughly 0.38–0.55 in the original equity/FX study, and some — like FX-trend at −0.40 — are outright negative alone). The app's whole thesis is that combining several sleeves that move independently produces a stronger combined result than any one of them, because portfolio Sharpe = single-sleeve Sharpe × √(number of uncorrelated sleeves). Each portfolio is just a named bundle of sleeves; the engine runs each as its own backtest, then risk-parity-combines them (weighting each by the inverse of its recent volatility, with no look-ahead) and scores the single combined return stream. The correlation matrix the app shows is the evidence — low, near-zero correlations mean the bets are genuinely independent (Dawnstar's sleeves average ~0.085 pairwise).

The seven strategies, in plain English

Trend is the unified core: applied to each instrument's OWN price, it measures whether price is above its level 3, 6, and 12 months ago (63/126/252 trading days), averages the three votes, and goes long when the trend is up, flat when down. Mean-Reversion is the equities-only 'buy the dip' sleeve: buy when the close is more than Buy% below its N-day moving average, sell when more than Sell% above, with a hard Stop% below entry — it has real knobs, so it is the only strategy the sweep and walk-forward can optimize. The anti-cheating fill rules

Four classic backtest lies are engineered out. (1) No look-ahead: a signal computed on day t's CLOSE fills at day t+1's OPEN — a --selftest mode proves this mechanically, and the build script runs it after every compile and aborts with 'do not trust results' if it fails. (2) Costs: adverse slippage (default 5 basis points) is applied against you on every fill, plus an optional per-order commission (default $0, matching Alpaca) — zero-cost backtests flatter dip-buying enormously, so this matters. (3) Adjusted data: the engine expects split/dividend-adjusted bars and warns on any >40% one-day move suggesting an unadjusted split. (4) Honest annualization: it only computes a CAGR when there are at least ~63 trading days, otherwise reporting the raw period return rather than inflating a short run.

Daily bars only: intraday fills, gaps through stops, and halts are NOT modeled; a stop 'fills' at the next open even if price gapped far through it — stress-test with Slippage bps 10 or 20.
Per-day FX financing is only modeled for the Carry and Blend sleeves (which auto-enable it), so a short-heavy FX trend run is slightly optimistic.

The Validation / Rigor Engine, in Plain English

Dawnstar's core differentiator is not a clever signal — it is a set of professional-grade anti-overfitting statistics, wired into the workflow so the app will tell you 'this isn't real' before you fund it. Everything rolls up to one tiered verdict.

The single verdict everything rolls up to

Instead of one impossible pass/fail bar, Dawnstar assigns a TIER, because a literal 3-sigma certainty on a single 10–20-year backtest is mathematically unreachable for any real strategy. INSUFFICIENT means fewer than 30 trades — statistically meaningless, treated as 'can't judge,' not a fail. REJECT (Tier C) falls short on significance and/or out-of-sample robustness — do not deploy real capital. PROMISING (Tier B) is real and robust but not yet 3-sigma proven: net out-of-sample Sharpe positive, a robustness measure passing the 'deploy-small' bar (Probability of Overfitting ≤ 0.50 OR ≥60% of cross-validation folds profitable), a t-stat ≥ 1.64, and at least half the required backtest length — the action is deploy small (10–25% of target risk), paper-track 6–18 months, and scale only if live tracks. PROVEN (Tier A) additionally needs a t-stat ≥ 3.0 (the Harvey-Liu multiple-testing bar), the stricter robustness (Overfitting ≤ 0.20 OR ≥75% of folds positive with a positive 5th-percentile fold), and enough data to clear the Minimum Backtest Length. Crucially, significance is judged on the CONSERVATIVE full-sample Sharpe, never the rosier fold mean.

The individual checks, in plain English

The Probabilistic Sharpe Ratio turns 'Sharpe 1.2' into 'X% confident the true Sharpe is above 0,' correcting for how short and skewed your sample is. The Deflated Sharpe Ratio measures that against the Sharpe you'd EXPECT to get by luck alone from N random trials — if your winner isn't clearly above what N coin-flips would produce, you found noise. The Probability of Backtest Overfitting (PBO), computed empirically via CSCV, splits history into 16 blocks and, across all 12,870 ways of choosing 8 for training and 8 for testing, asks how often the in-sample best lands below the median out-of-sample; above 50% means you are fitting noise (on the built-in sample strategy it reads 54%, a coin flip). Combinatorial Purged Cross-Validation (CPCV) is the check for a FIXED rule with no configs to sweep: it holds out every combination of history blocks (with an 'embargo' gap so a straddling position can't leak information) and reports the distribution of out-of-sample Sharpe. Minimum Backtest Length says how many YEARS you'd need so a Sharpe this high couldn't just be the luckiest of N random strategies. The Harvey-Liu haircut reports the raw t-stat and a deflated 'haircut Sharpe' against the ~3.0 bar. Monte Carlo resamples your trade sequence 1,000 times to show the RANGE of outcomes (5th/50th/95th percentile return and drawdown) — size for the p95, not the median. Walk-forward re-optimizes on prior data across (default) 4 folds and reports efficiency (out-of-sample Sharpe ÷ in-sample); near zero is the classic curve-fit signature.

Glossary — plain-English definitions

Sharpe ratio
A measure of return per unit of risk (volatility). Higher is better, but even great investors sit under ~0.8 over decades — so a backtest boasting 2–3 is a red flag, not a prize.
t-stat (t-statistic)
How many standard errors a result sits above zero — i.e. how confidently it is distinguishable from luck. Roughly Sharpe times the square root of years of data. Dawnstar treats ~3.0 as the bar for 'proven,' stricter than the usual 2.0.
PROVEN / PROMISING / REJECT / INSUFFICIENT (tiers)
Dawnstar's four verdicts. INSUFFICIENT = too few trades to judge. REJECT = fails the tests, don't risk money. PROMISING (Tier B) = real but not yet certain, deploy small. PROVEN (Tier A) = statistically strong, but still only on a backtest.
Tier A / Tier B (operating stance)
Tier A is a statistical badge a backtest can earn. Tier B is how you should actually behave with real money — deploy small, paper-track, scale only on forward confirmation. Dawnstar's rule: operate at Tier B even when the badge says Tier A, because a backtest has no live track record.
Sleeve
One strategy run on one asset class (e.g. 'trend on commodity ETFs'). Each sleeve is individually weak; the edge comes from combining several that move independently.
Breadth
Having many genuinely independent bets. Portfolio quality grows with the square root of the number of uncorrelated sleeves — so many modest, unrelated bets beat one clever signal.
Trend-following (time-series momentum)
Buy what has been going up, stay out of what has been going down, based on each instrument's own recent price. The one mechanism with decades of cross-asset evidence, so it is Dawnstar's core.
Mean-reversion
'Buy the dip' — buy when price falls well below its recent average, expecting a bounce. Dawnstar uses it only on equities and shelves it when a clear trend exists; never on crypto or FX.
Carry (FX carry)
Earning the interest-rate difference between two currencies (hold the high-yielder, short the low-yielder). It 'pays slowly and loses quickly,' so Dawnstar sizes it small and tail-aware, not as smooth income.
Low-vol (low-volatility)
Favoring the calmest instruments, which historically deliver better risk-adjusted returns than the wildest ones.
Deflated Sharpe Ratio (DSR)
A Sharpe corrected for how many strategies you tried. If you test 100 things and keep the best, the winner looks good by luck alone; DSR asks whether it beats what luck would produce.
Probability of Backtest Overfitting (PBO)
An empirical estimate (via CSCV) of how often your 'best' in-sample choice turns into a below-average performer out of sample. Above 50% means you are mostly fitting noise. The built-in sample strategy reads 54% — a coin flip.
CSCV / CPCV (combinatorial purged cross-validation)
Repeatedly splitting history into blocks, training on some and testing on the rest across every combination (with a gap so information can't leak), to see how a rule holds up out of sample rather than on one lucky split.
Monte Carlo
Reshuffling your trades 1,000 times to see the RANGE of possible outcomes instead of the single path you happened to get. Guidance: size for the unlucky 95th-percentile drawdown, not the median.
Walk-forward efficiency
How much of a strategy's in-sample performance survives when it is retested on later, untouched data. Near zero means it was curve-fit; above ~50% suggests a robust edge.
Minimum Backtest Length (MinBTL)
The number of years of history you'd need for a given Sharpe to be trustworthy at your trial count. A backtest shorter than this is flagged as indistinguishable from overfitting.
Trials (N)
The honest count of every strategy or variant you searched. The more you tried, the higher the bar to call the winner real — the most-omitted number in the industry, which Dawnstar makes first-class.
Probabilistic Sharpe Ratio (PSR)
The probability that a strategy's true Sharpe is above zero, adjusted for a short or lopsided track record — before the penalty for how many things you tried.
Kelly sizing
A formula for the mathematically 'optimal' bet size given a known edge. Dawnstar uses it only as a CAP (at most a quarter of it), never a target, because full Kelly over-bets when the edge is uncertain.
Risk parity
Weighting each sleeve by the inverse of its recent volatility so that calm and wild sleeves contribute roughly equal risk, rather than letting the wildest one dominate.
Non-custodial
Dawnstar never holds or moves your money. Your funds stay at your brokers; the app only sends orders through their APIs with keys you control.
Paper trading
Simulated trading against a practice account with fake money — the default mode, used to test everything safely before any real money is at risk.
Dry run
A cycle that computes and logs the exact orders it WOULD place but submits nothing. The GUI's dry-run button always previews only, even on paper.
Reconcile / target-and-reconcile
Dawnstar computes today's ideal positions ('targets') then trades only the difference to reach them, rather than stacking new orders on top of what you already hold.
The two live gates
Two independent switches (an environment flag LIVE_TRADING=true and a signed-off risk-acknowledgement file), both off by default and each armed with a single-use code, that together are required before any real order can be placed.
Nonce
A fresh, single-use confirmation code the app mints and you echo back to arm a live gate, so a captured code can't be replayed later.
Kill switch (STOP_TRADING)
An emergency stop — a file the engine and scheduler both honor — that halts all trading immediately, overriding strategy signals, schedules, and gates.
Argon2id / AES-256-GCM / DPAPI (the vault)
The encryption stack protecting your broker keys: a memory-hard password stretch (Argon2id), authenticated encryption (AES-256-GCM), and a Windows account binding (DPAPI). Same class of design as a local password manager; the master password is unrecoverable by design.
Survivorship bias
When a dataset includes only the winners that survived to today and silently omits the ones that died, making backtests look better than reality. A big reason Dawnstar sizes crypto and single stocks small.
Slippage
The gap between the price you expected and the price you actually got. Dawnstar's backtester charges an adverse ~5 basis points on every fill so results are never unrealistically cost-free.
Maker / taker fees
Exchange fees for adding liquidity (maker, cheaper) versus removing it (taker, dearer). On Coinbase retail these run roughly 0.6% / 1.2%, punishing enough that the engine models them and refuses trades that can't clear their cost.
PDT (Pattern Day Trader) guard
A US rule that limits accounts under $25,000 to three day-trades in five days. Dawnstar blocks a would-be fourth same-day round-trip (exits still allowed) to keep you compliant.
Annualization artifact
A measurement bug where a number is scaled up wrongly. Dawnstar's superseded 0.80 Sharpe came from counting crypto's weekend days as extra trading time; fixing it gave the honest, lower 0.75.
